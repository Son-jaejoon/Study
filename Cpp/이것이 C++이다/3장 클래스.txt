1. 클래스 : 함수를 포함할 수 있는 구조체의 확장, 접근제어지시자

2. 생성자와 소멸자 : 클래스 객체가 생성 및 소멸할 때 '자동으로' 호출되는 함수
- 배열을 new로 생성할 시, 배열로 delete를 해주어야 한다. (그렇지 않으면, 첫번째 요소만 소멸)
- 참조 형식 멤버일 경우 반드시 선언과 동시에 초기화 해야한다.
   (참조자를 단순 int로 생성자로 설정한다면, 쓰레기값이 저장된다 - 원본이 사라졌기 때문에)

- void fuct(void) = delete ; 실수로 사용할 수 있는 함수를 방지할 수 있다.(삭제된 함수 표시)

3. 메서드 : 함수 형태로 클래스의 실제 동작과 상태를 책임진다. 실제로 어떤 동작을 구현하는데
                   가장 중심이 됩니다.

4. this : 메서드 함수 내부에서 실제 클래스 인스턴스의 주소를 가리키는 포인터

5. 정적 멤버에서는 this를 사용할 수 없다.

6. 상수형 메서드 (int func() const {})는 쓰기(수정)을 할 수 없도록 한다.
- const의 위치는 함수 괄후 후에 해야한다. const int 라고 하면 반환값이 const int가 되기 때문에.
- 상수형 매개변수 (const int data)
- 이후 수정해야할 때 mutable 자료형 data / const_cast<자료형>(data)
